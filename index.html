Armazenamento de Documentos
Re-imaginado em Binário.

Baseado em `Markdown`. GoblinDB é um banco de dados **orientado a documentos** que utiliza uma arquitetura em camadas, o formato proprietário BMD (Binary Markdown) e o protocolo GBDB para garantir interoperabilidade total e armazenamento binário eficiente.

Ver no GitHub
$ docker pull makegoblindb/goblindb-server

Arquitetura em Camadas

Baseado em `ARCHITECTURE.md`. O GoblinDB adota uma separação rigorosa de preocupações, dividindo o sistema em camadas de Processamento de Consultas e Armazenamento Físico. Isso permite otimizações independentes e alta escalabilidade.

Camada de Processamento de Consultas

Gerencia autenticação de sessões, parsing do protocolo GBDB e execução do interpretador de consultas.

Camada de Armazenamento Físico

Gerencia o IO de arquivos BMD, manipulação binária direta (endianness) e o gerenciador de cache de blocos.

Fluxo de Dados

Cliente GBDB
Camada de Processamento (Parser GBDB)
Camada de Armazenamento (BMD IO)
Arquivo `.bmd` (Disco)

Formato BMD

Baseado em `BINARY-MARKDOWN.md`. Um formato de serialização de documentos que combina a expressividade do Markdown com a compactação do binário.

Protocolo GBDB

Baseado em `PROTOCOL.md`. O protocolo GBDB gerencia a troca de dados entre clientes e servidores, otimizando a interoperabilidade e garantindo que o transporte de dados binários BMD seja eficiente através da rede.

Ver Especificação do Cabeçalho (PROTOCOL-HEADER-APPENDIX.md)

Suíte de Testes Robusta

Baseado em `TEST-SUITE.md`. A integridade do GoblinDB é garantida por uma suíte de testes rigorosa que cobre todas as camadas do sistema, desde o parser BMD até a rede GBDB.

BMD Parser

Valida o parsing correto de todos os elementos BMD e a integridade do IO binário.

GBDB Protocol

Testa a conformidade do protocolo, manipulação de erros e o fluxo completo de requisição/resposta.

Concorrência

Testes de estresse para garantir a integridade dos dados sob alta concorrência e carga de rede.
